**WebSocket URL construction (SIG-001):**
Compute the WebSocket URL explicitly:
```javascript
const getWsUrl = () => {
  const protocol = window.location.protocol === "https:" ? "wss" : "ws";
  return `${protocol}://${window.location.host}/ws`;
};
```
This constructs `ws(s)://${window.location.host}/ws` which connects to the mediasoup signaling server at the `/ws` path (per SIG-001 decision). In production (behind nginx), this goes through the TLS-terminating proxy. In development, the Vite dev server needs a proxy config or the URL may need to point directly to the router server at port 3000.

Add to Channels.jsx: If `import.meta.env.VITE_ROUTER_WS` is set, use it as the base and append `/ws`. Otherwise use the window.location-derived URL. This matches the pattern in `getRouterWsUrl()` from `web-ui/src/utils/voicepingAudio.js` but targets the `/ws` path:
```javascript
const getWsUrl = () => {
  const envUrl = import.meta.env.VITE_ROUTER_WS;
  if (envUrl) {
    // envUrl may or may not have /ws, normalize
    return envUrl.endsWith("/ws") ? envUrl : `${envUrl}/ws`;
  }
  const protocol = window.location.protocol === "https:" ? "wss" : "ws";
  return `${protocol}://${window.location.host}/ws`;
};
```
